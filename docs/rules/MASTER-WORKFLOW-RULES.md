# Master Workflow Rules

## Purpose

These are my default workflow rules for future projects.

## Default rules

- Keep `.md` notes in the project repo
- Keep polished `.docx` reference copies in the Desktop `AI Rules Library`
- Let one AI tool drive a change at a time
- Use the rule creation process when adding new rules
- Commit workflow and rules changes when they affect the project

## Rule creation process

1. Name the problem or decision the rule is meant to solve
2. Decide what kind of rule it is
3. Choose the right file
4. Write the rule clearly
5. Write where it applies
6. Write why it exists
7. Add any exception if needed
8. Save the file
9. Commit it if it belongs in the project

## Desktop AI Rules Library rule process

When I create a new Desktop `AI Rules Library` rule for cross-project reference, I should do this by default:

1. Confirm that the rule is truly cross-project and not just for one repo
2. Add or update the Markdown source note in the Desktop `AI Rules Library`
3. If the rule changes my default operating system, add or update the master rules note
4. If helpful, add a short project-side mirror only when the current repo needs to follow that rule too
5. Refresh the matching `.docx` reference copy in the Desktop `AI Rules Library`
6. Keep the Desktop `.md` and `.docx` files out of the project repo unless there is a special reason
7. Commit only the project Markdown note if the project itself changed

Shortcut:

- cross-project rule = Desktop `AI Rules Library`
- project rule = project repo

## AI Rules Library content rule

The Desktop `AI Rules Library` folder should only contain universal AI project rules and closely related cross-project rule references.

That means it may contain:

- master workflow rules
- new project instructions
- rule creation guides
- Claude or other AI starter prompts
- cross-project AI workflow references

That means it should not become a storage place for:

- random files
- general tutorials with no rule value
- unrelated project documents
- one-off scratch notes
- files that do not help define or apply my reusable AI workflow system

Shortcut:

- if the file teaches or defines a reusable AI rule system, it can belong there
- if the file is random, one-off, or not about reusable AI rules, it should live somewhere else

## AI Rules Library vs AI Reference Library rule

Use the Desktop `AI Rules Library` for files that define, control, or apply my reusable AI workflow system.

Use the Desktop `AI Reference Library` for broader learning and reference materials that help me think, plan, or understand AI-related topics but do not act as core workflow rules.

`AI Rules Library` should hold things like:

- master workflow rules
- new project instructions
- rule creation guides
- AI starter prompts
- workflow-system references

`AI Reference Library` should hold things like:

- concept guides
- architecture references
- deployment references
- broader AI learning materials
- comparison documents

Shortcut:

- if the file tells me how my AI workflow system should operate, it belongs in `AI Rules Library`
- if the file teaches or explains useful AI topics more broadly, it belongs in `AI Reference Library`

## Starter reminder for new projects

Use the workflow rules in `AI-WORKFLOW.md` and the rule creation process in `HOW-RULES-WORK.md` unless I tell you otherwise.
